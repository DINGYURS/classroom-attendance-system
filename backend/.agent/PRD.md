# 产品需求文档 (PRD) 

## 一、产品概述

​	本产品旨在解决高校大班教学中传统点名方式效率低、代签现象严重及数据统计繁琐的问题 。 系统采用 **“端云结合”** 的模式：教师通过移动端/PC端拍摄或上传课堂合照（支持左中右多张拼接逻辑），后端利用计算机视觉技术自动检测并识别学生身份，实现“无感化”考勤。系统同时具备数据安全保护机制，支持考勤数据的可视化统计与导出 。

## 二、 目标用户

### 2.1 教师用户

- **场景**：上课前。
- **需求**：手机拍个照就能完成点名，能立刻看到谁缺勤，能导出Excel报表 。

### 2.2 学生用户

- **场景**：学期初录入信息、平时查看考勤。
- **需求**：简单的个人信息录入，考勤记录透明可查 。

## 三、 核心功能列表 

### 3.1 学生端

1. **账号登录**：学号+密码登录。
2. **人脸入库**：
   - 上传个人标准证件照/清晰生活照。
   - 系统需预检测（确保只有一张脸且清晰），提取特征并**加密存储**。
3. **个人中心**：
   - 查看个人考勤历史（出勤/缺勤/迟到）。
   - 修改学号、密码/更新人脸底库照片。

### 3.2 教师端 

1. **课程与班级管理**：
   - 创建课程，关联具体的教学班级（导入学生名单 Excel）。
2. **发起点名 (核心)**：
   - **合照上传**：支持上传 1-3 张照片（设计为左、中、右视角），适配移动端调用摄像头 。
   - **前端压缩**：上传前进行图片压缩（限制在 1600px 宽以内），加速传输。
3. **考勤结果确认**：
   - **可视化列表**：显示“已到名单”和“缺勤名单”。
   - **人工修正**：对于算法识别错误或漏检的学生，教师可手动修改状态（如将“缺勤”改为“请假”，或者是添加“迟到”）。
4. **数据统计**：
   - 按课程/班级查看出勤率折线图/饼图 (ECharts)。
   - 导出考勤报表 (Excel)。

### 3.3 算法与后台服务

1. **人脸检测与对齐**：使用 YOLOv11 检测合照中的所有人脸 。
2. **特征提取**：使用 InsightFace 提取 512维特征向量 。
3. **身份比对**：
   - Java 主导比对逻辑。
   - 计算余弦相似度，阈值暂定 0.6（可配置）。
4. **数据隐私**：
   - 数据库中 `feature_vector` 字段必须经过 AES 对称加密存储。

## 四、 功能优先级 

为了确保毕设能按时完成（MVP原则），建议按以下顺序开发：

| **优先级** | **功能模块**   | **说明**                                                     |
| ---------- | -------------- | ------------------------------------------------------------ |
| **P0**     | 基础人脸入库   | 学生上传照片，Python提取特征，Java加密存MySQL。              |
| **P0**     | 核心点名流程   | 教师上传1-3张图 -> Python返回特征 -> Java比对 -> 生成考勤记录。 |
| **P0**     | 结果展示与修正 | 简单的列表展示缺勤/出勤，支持手动修改状态。                  |
| **P1**     | 移动端适配     | 使用 Element Plus 响应式布局，适配手机浏览器拍照。           |
| **P1**     | 数据可视化     | ECharts 展示出勤率饼图、趋势图。                             |
| **P1**     | Excel 导入导出 | 批量导入学生名单，导出期末考勤表。                           |
| **P2**     | 标注检测框     | 前端在图片上画框显示识别出的名字（工作量大，最后做）。       |
| **P2**     | 缺勤预警       | 缺勤超过3次自动标红或发送通知 8。                            |

## 五、界面设计

### 5.1 整体风格

- **配色**：简洁、学术风（蓝、白、灰为主）。
- **组件库**：Element Plus。

### 5.2 关键页面草图

1. **教师-发起点名页（移动端优先）**：
   - 顶部：课程名称、班级下拉框。
   - 中部：三个巨大的“+”号方块，分别标记“左侧视角”、“中间视角”、“右侧视角”。点击唤起摄像头。
   - 底部：“开始点名”大按钮（Loading 状态需明显）。
2. **教师-结果确认页**：
   - 上半部分：缺勤人数/总人数概览卡片。
   - 下半部分：缺勤学生名单（卡片式布局，包含头像、姓名、学号、操作按钮“标记为已到”）。
3. **学生-人脸采集页**：
   - 中心是一个圆形取景框，提示“请正对摄像头，保持光线充足”。

## 六、技术栈建议 

- **前端**：Vue 3 (Script Setup) + Element Plus + Axios + ECharts+ Tailwindcss + **compressorjs** (图片压缩)。
- **后端**：Spring Boot 3 + MyBatis + Hutool (工具类/加密)。
- **算法微服务**：Python 3.11 + FastAPI + PyTorch + **YOLOv11** + **InsightFace**。
- **数据存储**：
  - 结构化数据：MySQL 8.0 (使用 **JSON** 类型存特征，开启 AES 加密)。
  - 非结构化数据：MinIO (存储原始图片)。
- **部署运维**：Docker + Docker Compose + Nginx (反向代理)。

## 七、代码风格与架构模式

### 7.1 架构模式

- **服务解耦**：
  - **Java (Core)**：负责用户认证、业务逻辑、数据库读写、特征比对、加密解密。
  - **Python (Worker)**：无状态服务。**Input**: 图片URL -> **Output**: 人脸坐标 & 特征向量列表。不连接 MySQL。

### 7.2 代码风格

- **Java**: 遵循《阿里巴巴Java开发手册》。Controller 层只做参数校验和转发，业务逻辑下沉到 Service 层。
- **Python**: 遵循 PEP8 规范。使用 Type Hints (类型提示) 增加代码可读性。
- **Vue**: 使用 Composition API (`<script setup>`)，逻辑复用使用 Hooks。

### 7.3 API 规范

- 遵循 RESTful 风格。
- 统一响应结构：`{ code: 200, message: "success", data: ... }`。
